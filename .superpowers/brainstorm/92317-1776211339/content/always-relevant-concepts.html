Always Relevant Section — 3 Concepts

This section needs to support both websites AND apps. Here are three ways to handle it.

A Tabbed Sites / Apps

Two tabs within the section — "Sites" and "Apps". Clean separation, each with its own add input. Matches the profile detail pattern.

  • Same add pattern as blocking profiles

Tradeoffs

  • Can't see sites + apps together
  • Extra click to switch between
B Side-by-Side Columns

Sites on the left, apps on the right. See everything at once. Each column has its own "+ Add" link.

C Mixed Chips (Recommended)

Sites and apps live in one chip list. Each chip has a small type indicator. One add input with a site/app toggle. Simplest, most compact.

Strengths

  • Most compact — everything in one flow
  • Type labels ("app") make it clear what's what
  • One input handles both (toggle switches context)
  • Matches the simplicity of the drill-down list page

Tradeoffs

  • Mixed types in one list can feel less organized with many items
  • Toggle adds a small learning curve

My recommendation: C (Mixed Chips)

Most people will have 3-8 always-relevant items total. A single chip list with type labels keeps it compact and scannable. The site/app toggle on the input is a clean way to handle both types without doubling the UI. It also keeps the Distractions list page clean — one simple section at the bottom.
