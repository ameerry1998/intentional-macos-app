#### Bugs reported during testing
- [x] Bedtime/Wake bands showed default 22:00/07:00 even when nothing configured — fixed

- [ ] **Settings page is "busy"** — duplicates content that now lives in Sensitive Content / Accountability sidebar items. Needs cleanup pass: strip the duplicated sub-sections from Settings, leave only true config (account, app prefs, bedtime, strict mode, distractions list management).
- [x] **Strictness change didn't persist** (commit `19472f7`) — clicking Strict + Done left the value lost. Root cause: legacy UPDATE_PROJECT path doesn't carry strictness_preset, and the proper dedicated endpoint (PUT /intentions/{id}/strictness) was never called. Fixed: dashboard now sends UPDATE_INTENTION_STRICTNESS as a separate message when strictness changes; MainWindow.handleUpdateIntention also detects diffs and calls IntentionStore.updateStrictness. **CONFIRMED WORKING 2026-05-08:** tightening to Strict persists end-to-end (backend + dashboard re-render after navigation back). Softening (Strict → Standard) NOT yet tested — see §1.5 follow-ups.
- [x] **Crash: `-[__NSCFNumber count]` in `MainWindow.uiPerfMaybeFlush`** — heartbeat-driven `EnforcementReconciler.pushStateToDashboard → callJS` mutated debug perf counters off-main while `userContentController(_:didReceive:)` mutated them on main. Race corrupted the dictionary's heap storage and triggered abort. Fixed: counter mutations + `uiPerfMaybeFlush` now run inside the same `DispatchQueue.main.async` block as `evaluateJavaScript`. Rule: never mutate Swift Dictionary or Array off-main when another thread also writes it.
- [x] **`openIntentSetup` lost strictness/budget/apps on reopen** — when clicking an existing Focus Mode card the editor seeded `state.selected` from `intentionsCache` but only copied id/name/description/color. `strictness_preset`, `weekly_budget_hours`, `mac_websites`, `mac_bundle_ids` were dropped, so the form always reseeded as Standard. Fixed in `dashboard.html`: `openIntentSetup` now copies all backend fields and maps `mac_websites` + `mac_bundle_ids` into the allowed list shape the form expects.
- [x] **Draft blocks autosaved before Done** (commit `910cc55`) — picking a Focus Mode in the editor for a new block fired markFocusDirty → SET_SCHEDULE → block became active in Swift BEFORE user clicked Done, so distractions got blocked while editor was still open. Root cause: draft pattern was honored in addFocusBlock/onCalendarHourClick (no save on creation) but `onEditorIntentionChange` and the day-pill click handler called markFocusDirty unconditionally. Fixed by gating those inline saves on `!focusState.draftBlockId`. Saved on Done via saveBlockEdit() unchanged.
- [x] **Focus Mode editor missing fields + legacy concept bleed** (commits `4dd1b5b`, `3d49cfd`) — title said "Edit project", no strictness picker UI, no weekly target hours input, legacy "Blocklists" pulldown still rendering, "Blocked hosts" / "Allowed hosts" copy didn't communicate per-block overrides. Fixed: renamed all "project"→"Focus Mode", added Strictness dropdown (Strict/Standard/Soft), added Weekly target hours number input, removed legacy Blocklists pulldown, added "How rules layer" explainer card framing the relationship between global Distraction Budget and per-Focus-Mode override rules. Spec §4 updated.

#### Slice 13 fixes — verified 2026-05-13
Three uncommitted fixes on `slice-13-cleanup` passed manual testing today against the DerivedData Debug build:
- [x] **AI scoring gate for manual Focus Mode sessions** (`FocusMonitor.swift:~1679`). Removed `guard state.isWork else …` — manual sessions started from a Focus Mode card don't have a corresponding ScheduleManager block (they live only in `/focus/active`), so `state.isWork` was false and the scorer was being skipped entirely. Now `focusModeController.isOn` is the canonical gate. Verified: starting a coding-mission session and visiting YouTube triggered the in-pill "Not related to your task" nudge.
- [x] **Comet / ChatGPT Atlas tab inspection during Focus sessions** (`BrowserMonitor.swift` + `FocusMonitor.swift`). Both were on the excluded-bundles list as "AI assistants" — but they're full Chromium browsers that load arbitrary websites. Moved them to `knownBrowserInfo` + `FocusMonitor.browserBundleIds`. Also added Dia and corrected real bundle IDs for SigmaOS / Wavebox / Sidekick; removed phantom `nickvision.*` entries that never matched real installs. Verified: boot log shows `updateBlockedBrowsers: ["Chrome", "Arc", "Comet"]` and `🔍 Comet: Found 23 tabs`.
- [x] **Draft block click-away no longer silently discards user work** (`dashboard.html:~10056`). Previously: click outside an in-progress draft block destroyed it without confirmation. Now: if title is non-empty, click-away saves (treated as implicit Done); empty title still discards silently. Existing-block click-away behavior unchanged.
