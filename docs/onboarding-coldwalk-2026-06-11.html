Onboarding cold-walk — what a brand-new user actually sees

2026-06-11 · Fresh signup (ameer.rayan+coldwalk@gmail.com) in the Tart VM, driven headlessly via the DEBUG ui-test hook. Screenshots captured per screen. Build: post-rebrand main (056ff31).
Verdict: the auth flow is launch-ready; the onboarding wizard is not. Screens 1–2 of auth (email → code) are clean and on-brand. The 5-screen wizard after it is a previous generation of the product: violet/indigo styling from before the coral design system, a theme picker for a dark-only app, a "platforms" screen configuring features that were deleted with the browser extension, and partner + settings-lock asks placed before the user has experienced any value — the exact inverse of the agreed value-before-commitment design. It reads vibecoded because it is: it was built for a different app and never revisited.
