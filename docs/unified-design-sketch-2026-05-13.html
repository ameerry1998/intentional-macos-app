Unified Design — taking Opal's simplicity into Intentional

A sketch of how to fold Opal's blocker feel, Unbed's wake ritual, Covenant Eyes' accountability, and our planning system into one coherent product surface. The goal: drop the sidebar from 5 items to 3, make Today look like "Now + Next + Plan + Status," and stop hiding our differentiators on pages users never visit.

Process used to build this sketch

  1. Tear down each reference app. What does Opal / Unbed / Covenant Eyes do better than us, and what specifically would we copy?
  2. Audit what our app gets wrong today. Where is it busy, redundant, or burying our best features?
  3. Identify the unifying moves. Two or three architectural changes that reduce surface area without losing capability.
  4. Sketch the resulting screens, starting with the one the user lives in (Today), then the configuration surface (Settings). Plan + Wake follow in their own sketches.

What our current app gets wrong

  • Five sidebar items (Today, Focus Modes, Sensitive Content, Accountability, Settings). Opal does it with 2. Sensitive Content holds a single toggle and still gets its own tab.
  • Focus Modes are configurations, not missions. A grid of named cards with no schedule attached forces a clunky two-step "create the mode, then go elsewhere to schedule it."
  • Today has four competing sections. Coach Context + Schedule + Focus Mode toggle + AI Assessments. None is the headline. Opal's "Now" card IS the headline.
  • Our differentiators are buried. Sensitive Content and Accountability — the two things Opal can't do — are tucked away on separate pages. They should be visible always, even if just as a status row.
  • No "what's active right now" surface. A new user opens our app and can't tell at a glance if they're in a session, if a partner is connected, if content protection is on, or what's next. Opal answers all of that in the first card.

The unifying moves

MoveWhy
1. Collapse sidebar to 3 items. Today / Plan / Settings. Focus Modes folds into Plan (they become missions tied to goals). Sensitive Content + Accountability fold into Settings as rows in a "Defenses" section. Their status becomes pills in a status footer on Today.
2. Today = "Now + Next + Plan + Status." Opal-shaped page. One loud Now card. One Up-Next row. Quick-action buttons. Today's missions as a list. Status footer pinning Partner + Content + Puck as always-visible.
3. Plan absorbs Focus Modes. You don't create "modes" in isolation. You create missions in Plan, give each a strictness, drop them onto today's timeline. Same surface, simpler concept, fewer steps.
4. Defenses earn an always-on status row. Caity · partner active — Content protection · ON — Puck · paired. Visible on every Today view. The user is reminded what's running without navigating to verify.

What's still to sketch

This sketch covers Today + Settings. The next sketches should cover, in priority order:

  • The Plan page — already spec'd in docs/superpowers/specs/2026-05-13-planning-system-spec.md, but should be re-sketched in this same visual language so all three tabs feel like one product.
  • The wake-up flow — alarm fires → phone shows "Tap Puck to wake" → after tap, transition into the planning ritual. Spec after the Cat 4 Perplexity research on Unbed + AlarmKit lands.
  • The drift redirect overlay — when AI detects off-mission activity. Currently an external nudge; should feel more like a coach message, less like an interrupt.
  • The end-of-session card — already exists ("Block Complete" + carousel) but needs to be re-sized against this new visual language.
Sketch, not spec. Don't build from these pixels. The intent is to lock the architecture (3-tab sidebar, "Now + Next + Plan + Status" on Today, defenses folded into Settings, status pills always visible) and the shape of each surface. Visual polish — typography scale, exact spacing, motion, color refinement — comes in a real design pass against the architecture this sketch defines.
