Focus concepts —
the simplification

We have nine concepts doing one feature's worth of work. Every bug we hit tonight came from these concepts disagreeing with each other. Here's the proposal to collapse them.

No code changes proposed. Read first, redline, then we plan.
The problem

Nine concepts doing one feature's job.

If I asked you "is focus mode on right now?" — there's no single answer. There are nine separate pieces of state that each have an opinion. They drift apart. Bugs follow.

The proposal

One concept — Focus Mode — with three states.

Three states is the floor. Anything fewer can't capture "I'm focusing" vs "I'm sleeping." Intention is just metadata on FOCUS — not a separate state.

OFF
Free time
Mac is a Mac. iPhone is an iPhone. No enforcement, no overlays, no AI scoring, no red shift, no blocking. The default state when you wake up and the default state late at night.
No blocking No AI scoring No overlays
FOCUS
"I'm focusing"
The full enforcement bundle is live. Optionally has an intention attached ("writing email to X"); intention is metadata, not a separate state. Triggered by puck tap, Cmd+Shift+P, scheduled block start, or manual toggle.
All FOCUS features Optional intention
→ See the Feature inventory below for what's in the bundle.
BEDTIME
Sleeping isn't focusing
Separate from FOCUS because the goal is different — wind down, not produce. Different blocklist (social, news, stimulating). Triggered by sleep-time or manual.
Bedtime blocklist Wind-down ritual
Feature inventory

Every feature, grouped by which state it runs in.

Three buckets: features tied to FOCUS, features tied to BEDTIME, and the "backbone" that runs regardless of state (security, persistence, sync infrastructure).

⏱ FOCUS state — the focus enforcement bundle

All of these turn on together when Focus Mode flips to FOCUS. None of them run when Focus Mode is OFF.

  • App blocking — distracting apps shut
  • Site blocking — DNS + browser overlay
  • Switch overlay — 10s countdown on app switch
  • AI relevance scoring — tabs scored vs intention
  • Grayscale / red shift — when off-task
  • Pill widget — time, score, current state
  • Off-task nudges — levels 1 → 3 escalation
  • Block start ritual — at scheduled block start
  • Block end ritual — at scheduled block end
  • Time tracking — focused / off-task minutes
  • Earned browse pool — accrue + spend
  • Cross-device mirror — iPhone FamilyControls fires too

🌙 BEDTIME state — wind-down enforcement

Different goal (sleep, not focus) → different bundle. Most FOCUS features are off here.

  • Bedtime blocklist — social, news, doom-scroll
  • Wind-down ritual — gradual UI dim
  • Pill in bedtime mode — different visual
  • Wake alarm prep — iPhone alarm primed
  • Earned browse paused
  • No AI scoring — you're not focusing

🛡️ Backbone — runs regardless of state

These don't care whether Focus Mode is OFF / FOCUS / BEDTIME. They're security and infrastructure that's always on.

  • Strict mode persistence — app watchdog, login item
  • Tamper detection — CS state divergence
  • Content safety — NSFW screen capture
  • Partner lock — server-side enforced
  • Always-allowed apps — whitelist that never blocks
  • Cross-device sync infra — WebSocket, auth refresh
  • Schedule storage — blocks live in DB
  • Auth + tokens — Keychain, refresh

🤔 Decide where these belong

Features that don't cleanly fit one bucket. We'd nail these down before implementing.

  • "Plan-your-day" lock screen (the original Focus Gate). Triggered when you're in scheduled work hours but Focus Mode is OFF. Could be: separate optional nag, OR auto-flip Focus Mode ON when wake time hits + show planning prompt. Q1 in open questions.
  • Distractions list editing. Shared between Focus and Bedtime, but different lists per state. UI lives in Settings; data is account-scoped.
  • Schedule itself. Calendar UI lives outside Focus Mode; it just triggers state changes.
How it turns on/off

One rule: anything you tap, swipe, or schedule maps to this.

No more "did the puck tap fire a session AND a focus block? does the schedule agree?" — every input is a trigger that flips Focus Mode.

What gets renamed or removed

The mapping from old to new.

Focus Gate→ Focus Mode (or kept as optional "auto-lock if work hours and no plan" sub-feature)
Intentional Mode→ collapsed into Focus Gate
Focus session→ "Focus Mode ON with intention" (metadata, not a separate concept)
Always-active blocking→ "ON (open) blocklist"
Per-session profile→ "intention-specific blocklist" layered on top
TimeState enum (6 values)→ OFF · FOCUS · BEDTIME
iOS "Focus mode"→ "Intention" (a saved set of blocks + name; populates Focus Mode ON when activated)
unplanned enforcement path→ deleted. Unplanned = OFF.
What stays unchanged

Genuinely separate concepts.

Not everything collapses. These are orthogonal and stay where they are.

  • The schedule (calendar view, blocks). Becomes a trigger source instead of a separate enforcement engine.
  • Bedtime mode. Sleeping isn't focusing — different feature.
  • Always-allowed apps. Apps you can use even in Focus Mode (Slack, calendar, etc.).
  • Partner lock. Cross-device security — orthogonal.
  • AI relevance scoring. Runs whenever Focus Mode is ON, dormant otherwise.
  • The pill widget. Renders the current Focus Mode state.

Open questions

Decide together when you wake.

Q1
Should "no plan during scheduled work hours" still trigger a screen lock? That's the original Focus Gate. Feels like a separate optional nag feature on top of Focus Mode rather than core.
Q2
Auto-on at wake time? "Focus Mode is ON by default during waking hours" feels aggressive. Most people want to ease in. Default: stays OFF, you flip it on each morning.
Q3
What happens on Mac when you tap puck on iPhone? Focus Mode ON. Should it default to "open" or use the iPhone Mode's name as intention? Probably the latter.
Q4
iOS-only device (no Mac) — Focus Mode lives on iPhone alone. Sync to a non-existent Mac is a no-op until they install Mac app. Acceptable.
What this fixes

Every bug we hit tonight, in one stroke.

  • Phantom session. No separate "session" object — can't drift from "is focus on."
  • Stale active session in backend. Periods have explicit start/end; auto-close on missed heartbeat.
  • unplanned = enforcement. Unplanned = OFF. AI scorer never fires when no plan + no session.
  • Confused Focus Gate toggle. Toggle directly maps to ON/OFF; visible enforcement starts immediately or doesn't.
  • "iPhone says ended, Mac says active." Both apps subscribe to the same Focus Mode state in the backend.
  • Screen turns red on YouTube without an active session. No Focus Mode ON → no AI scoring → no red shift.
What this doesn't fix

Separate work that stays orthogonal.

  • iPhone fire-and-forget POST. Still need persistent retry queue on iPhone.
  • Schedule UI on iPhone. Still its own port.
  • Distractions sync (account-scoped Mode metadata). Still its own feature.

These each become smaller once Focus Mode is the only thing.
